# MTS True Tech Hack 2026 - LocalScript

Локальный AI-агент для генерации, доработки и отладки Lua-кода без внешних LLM API.

## О проекте

Этот репозиторий содержит наше решение для хакатона MTS True Tech Hack 2026 (трек LocalScript).  
Цель трека - сделать агентскую систему c слеюущими качествами:

- работает полностью локально;
- принимает задачу на естественном языке;
- генерирует корректный Lua-код;
- умеет уточнять требования и итеративно улучшать результат.

Подробное ТЗ: `docs/HACKATHON_SPECS.txt`.


## Стек

- LLM: `qwen2.5-coder` через локальный `Ollama`
- Backend API: `FastAPI` (Python)
- Проверки кода: `luac`, static guard, опционально линтер и семантическая валидация
- Оркестрация и доставка: `Docker Compose`
- UI и demo-клиенты: `Streamlit` + CLI

## Ключевые возможности

- `POST /generate` - генерирует Lua-код по текстовому запросу. Если постановка недостаточно конкретна, сервис возвращает уточняющий вопрос (`clarification`), а после уточнений выдает готовый код (`code`).
- `POST /refine` - принимает историю доработок и текстовый фидбек к предыдущей версии, затем возвращает обновленный вариант кода с тем же пайплайном проверок.
- `POST /debug` - анализирует пользовательский Lua-код, запускает локальные проверки, формирует описание проблемы и предлагает исправленный вариант.
- `GET /health` - показывает статус API, доступность Ollama, готовность модели и ключевые инференс-параметры.
- Repair loop: автоматические проверки и повторные попытки исправления до выдачи результата.
- Прозрачная диагностика: в ответах сохраняется история попыток и результаты проверок.

Полный контракт API: `docs/API_CONTRACTS.md` и `localscript-openapi.yaml`.

## Результат на хакатоне

- Команда из 2 человек
- 32 место из 132 команд

Были выполнены все основные условия хакатона - локальный воспроизводимый запуск, итеративная работа над кодом, удобынй интерфейс для использования агентской системы.

Кроме того, для повышения стабильности корректных ответов модель самостоятельно тестирует качество сгенерированного кода и при необходимости исправляет его с использованием результатов тестирования.
