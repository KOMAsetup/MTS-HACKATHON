# Проверка пиковой VRAM (условие жюри)

Цель: убедиться, что при инференсе с **`num_ctx=4096`**, **`num_predict=256`** пик памяти **≤ 8 GB** и вычисления на GPU (без CPU offload весов).

## Шаги

1. Запустите стек: `docker compose up` (или локальный Ollama + приложение).
2. Подгрузите модель: `ollama pull qwen2.5-coder:7b` (или ваш зафиксированный тег).
3. В отдельном терминале запустите мониторинг:

```bash
watch -n 0.5 nvidia-smi
```

4. Выполните несколько тяжёлых запросов (длинный `prompt` + большой `context` из `benchmarks/public_tasks.json`).

5. Зафиксируйте **максимальное** значение `Memory-Usage` / `MiB` в `docs/hardware.md` или в `docs/experiments/BEST_CONFIG.md` вместе с датой и тегом модели.

## Переменные

- В контейнере Ollama: `OLLAMA_NUM_GPU=999` (в `docker-compose.yml`) — ориентир на загрузку слоёв на GPU; уточняйте по документации вашей версии Ollama.

Если пик > 8 GB: уменьшить модель/квант или сменить тег; **не** менять `num_ctx` / `num_predict`, если жюри проверяет фиксированные значения.
